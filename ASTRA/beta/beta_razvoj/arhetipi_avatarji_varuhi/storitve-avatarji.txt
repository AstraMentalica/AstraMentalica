Pri listenerjih nisem mislil:

avatar_nekaj.php
avatar_tretje.php

ampak logiko znotraj listenerjev.

Primer:

class PoslusalecUporabnikRegistriran
{
    public function obdelaj(...)
    {
        avatar_dodaj_tocke(...);
    }
}

Taka datoteka se ne briše.

Briše ali prestavlja se samo ta del:

avatar_dodaj_tocke(...)

ker je to poslovna logika uporabnika.

Listener pa lahko ostane.

Pravzaprav bi jaz naredil takole:

Faza 1

Pusti:

kernel/poslusalci/*

pri miru.

Zabeležka:

PREVERITI KASNEJE
Faza 2

Ko prideva do:

SISTEM/storitve_svetov/uporabniki/

takrat bova našla:

avatar_dodaj_tocke()
avatar_dodaj_xp()
avatar_odkleni(...)

in se odločila:

ostane
premakni
razdeli

Trenutno pa si po mojem naredil zelo dobro čiščenje:

✓ runtime/kanali odstranjeni

✓ odziv_ai odstranjen
✓ odziv_api odstranjen
✓ odziv_web odstranjen

✓ adapter_odziv ostane
✓ adapter_izhod ostane
✓ adapter_napake ostane

✓ kernel še nedotaknjen

To je varen poseg.

Listenerji pa niso varen poseg, dokler ne vidiva dejanske kode.
