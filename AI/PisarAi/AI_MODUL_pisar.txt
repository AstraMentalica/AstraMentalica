Za več modulov z ločenimi AI agenti potrebuješ skupni okvir,
da bo integracija enostavna. Enotni API standardi naj pokrivajo:

1. Avtentikacija in dostop
– OAuth2 ali API ključ na vsaki zahtevi.
– HTTPS za vse klice.

2. Struktura zahtevka
JSON kot univerzalni format.
Primer:

{
  "module":"text_writer",
  "action":"generate",
  "payload":{
    "topic":"AI in varnost",
    "length":"long",
    "language":"sl"
  }
}

– module: identifikator modula.
– action: operacija (generate, analyze, fetch…).
– payload: parametri specifični za modul.

3. Struktura odgovora

{
  "status":"ok",
  "data":{...},
  "meta":{
    "tokens_used":123,
    "timestamp":"2025-09-29T20:15:00Z"
  },
  "errors":[]
}

– Vsi moduli vračajo status, data, meta, errors.
– Enotna obravnava napak (koda + opis).

4. Verzije
– v1 v URL-ju: /api/v1/text_writer/generate.
– Dodatne verzije ne rušijo obstoječih modulov.

5. Logika modulov
– Vsak modul izpostavi iste osnovne metode: status, config, execute.
– Izhodni format vedno JSON s polji result in metrics.

6. Razvojna pravila
– PSR-12 za PHP ali PEP-8 za Python.
– Vsa sporočila in nastavitve v slovenščini, rezervirane besede ostanejo angleške.
– Šumnike lahko transliteriraš po tvojem pravilu (s->sc, č->c…).

Tvoj prvi modul (AI pisec) se temu prilagodi:
– endpoint npr. /api/v1/text_writer/generate
– action generate, payload z naslovom teme in dolžino.
– Odgovor vsebuje generirano besedilo, porabo tokenov in morebitne napake.

S tem imajo vsi prihodnji moduli (npr. “monitor strani”,
“SEO analizator”, “cron upravljalnik”)
enako vstopno točko in format.
